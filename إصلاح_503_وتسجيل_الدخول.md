# إصلاح 503 وخطأ تسجيل الدخول (JSON.parse)

ظهور **503** عند الدخول عبر جوجل و**خطأ JSON.parse** عند الدخول بالبريد يعني أن طلبات **/api** لا تصل إلى تطبيق Node، أو أن التطبيق لا يعمل على المنفذ **5000**.

اتبع الخطوات التالية من **SSH** بالترتيب.

---

## 1) التحقق من أن Node يعمل ويستمع على المنفذ 5000

نفّذ:

```bash
curl -s -o /dev/null -w "%{http_code}" http://127.0.0.1:5000/api/auth/user
```

- إن ظهر **200** أو **401** أو أي رقم (وليس خطأ اتصال): تطبيق Node يعمل، والمشكلة غالباً من إعداد Apache (انظر الخطوة 3).
- إن ظهر **Empty reply** أو **Connection refused** أو لم يظهر رقم: تطبيق Node إما متوقف أو لا يستمع على 5000 → انتقل للخطوة 2.

تحقق أيضاً من وجود عملية Node:

```bash
ps aux | grep index.cjs
```

إن لم يظهر أي سطر فيه `index.cjs` فالتطبيق غير مشغّل.

---

## 2) تشغيل تطبيق Node من مجلد public_html

بما أنك فككت **node_app_upload.zip** داخل **public_html**، التطبيق يُشغّل من هناك. تأكد من وجود ملف **.env** في **نفس مجلد index.cjs** (أي داخل public_html):

```bash
cd /home/894422.cloudwaysapps.com/cmkdrtgqcv/public_html
ls -la .env index.cjs
```

**إن لم يكن .env موجوداً** أنشئه:

```bash
cd /home/894422.cloudwaysapps.com/cmkdrtgqcv/public_html
printf '%s\n' 'NODE_ENV=production' 'PORT=5000' 'HOST=127.0.0.1' 'SESSION_SECRET=غيّر-هذه-السلسلة-العشوائية-الطويلة' 'ADMIN_EMAIL=arb-33@hotmail.com' 'DATABASE_URL=file:sqlite.db' > .env
```

(عدّل **SESSION_SECRET** و **ADMIN_EMAIL** إن رغبت.)

ثم أوقف أي عملية Node قديمة وشغّل من جديد:

```bash
pkill -f "node index.cjs"
nohup node index.cjs > app.log 2>&1 &
```

انتظر ثانيتين ثم اختبر:

```bash
curl -s http://127.0.0.1:5000/api/auth/user
```

يُفترض أن ترى `null` أو رسالة JSON (وليس صفحة HTML أو خطأ اتصال).

---

## 3) التحقق من توجيه Apache إلى المنفذ 5000

يجب أن يكون في **public_html** ملف **.htaccess** يوجّه **/api** إلى **http://127.0.0.1:5000**. تحقق:

```bash
cat /home/894422.cloudwaysapps.com/cmkdrtgqcv/public_html/.htaccess | head -20
```

يجب أن ترى شيئاً مثل:

```apache
RewriteRule ^api(.*)$ http://127.0.0.1:5000/api$1 [P,L]
```

وقبلها (لتسجيل الدخول مع HTTPS):

```apache
RequestHeader set X-Forwarded-Proto "https"
```

إن كان **.htaccess** غير موجود أو لا يحتوي على القاعدة أعلاه، أعد رفع **public_html_upload.zip** (الذي فيه .htaccess الصحيح) أو أضف السطور يدوياً كما في ملف **حل_مشكلة_تسجيل_الدخول.md**.

على Cloudways تأكد أن **mod_proxy** و **mod_proxy_http** مفعّلان (غالباً مفعّلان افتراضياً).

---

## 4) بعد الإصلاح

- جرّب تسجيل الدخول بالبريد وكلمة المرور من **https://sharfedu.com/login**.
- جرّب الدخول عبر جوجل.

إن استمر 503: راجع سجلات Apache على السيرفر (موضعها يعتمد على إعداد الاستضافة) أو تواصل مع دعم Cloudways واذكر أن طلبات **/api** يجب أن تُوجّه إلى **127.0.0.1:5000**.
