# إعداد Node وإصلاح الثغرات على السيرفر

## ما تم في المشروع (المحلي)

- **npm audit fix:** تم تطبيقه؛ تم إصلاح ثغرة lodash.
- **nodemailer:** تم ترقيته من 6.x إلى 8.x لإصلاح ثغرات معروفة.
- **المتبقي:** ثغرة في **pm2** (لا يوجد إصلاح من الحزمة حتى الآن) وتبعيات **tar** داخلية — لا تؤثر على تشغيل الموقع في الاستخدام العادي.

---

## إصدار Node على السيرفر (لإزالة التحذيرات)

السيرفر حالياً يستخدم **Node v18**. بعض الحزم (مثل Vite) تطلب Node 20+ فتَظهر تحذيرات `EBADENGINE` عند `npm install` — لكن التطبيق يعمل لأننا لا نشغّل البناء على السيرفر.

**إذا أردت إزالة التحذيرات واستخدام Node 20:**

### الطريقة 1 — من لوحة Cloudways

1. ادخل إلى **Cloudways** → اختر السيرفر ثم التطبيق.
2. ابحث عن **Application Settings** أو **PHP & Node Version** أو ما شابه.
3. إن وُجد خيار **Node.js version**، غيّره إلى **20.x** واحفظ.

### الطريقة 2 — عبر SSH (إن كان nvm مثبتاً)

نفّذ في الطرفية:

```bash
# التحقق من وجود nvm
command -v nvm || [ -s "$HOME/.nvm/nvm.sh" ]

# تحميل nvm وتثبيت Node 20
. "$HOME/.nvm/nvm.sh" 2>/dev/null || true
nvm install 20
nvm use 20

# التحقق
node -v
# يفترض أن يظهر v20.x.x

# ثم من مجلد التطبيق أعد تثبيت الحزم
cd /home/894422.cloudwaysapps.com/cmkdrtgqcv/git_repo
npm install
```

**ملاحظة:** إذا كان الـ cron أو تشغيل التطبيق يعتمدان على مسار `node` ثابت، قد تحتاج بعد التبديل إلى Node 20 أن تضبط المسار في السكربت أو الـ cron لاستخدام نفس إصدار Node (مثلاً `~/.nvm/versions/node/v20.x.x/bin/node`).

## ملخص

| البند | الحالة |
|-------|--------|
| ثغرات قابلة للإصلاح (lodash, nodemailer) | تم إصلاحها في المشروع |
| تحذيرات Node (EBADENGINE) | اختيارية: ترقية السيرفر إلى Node 20 لإزالتها |
| pm2 / tar | لا إصلاح متاح حالياً؛ لا يمنع التشغيل |

بعد أي تعديل في `package.json` أو `package-lock.json` ارفع التغييرات ثم على السيرفر نفّذ `git pull` (أو انتظر النشر التلقائي) ثم `npm install` من داخل `git_repo`.
