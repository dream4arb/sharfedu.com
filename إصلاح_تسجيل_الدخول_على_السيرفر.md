# إصلاح تسجيل الدخول (التقليدي + جوجل) على السيرفر

إذا كان تسجيل الدخول التقليدي أو عبر جوجل لا يعمل بعد النشر، تحقق من التالي **على السيرفر**.

---

## 1) ملف `.env` في مجلد تطبيق Node

التطبيق يقرأ `.env` من **نفس المجلد الذي فيه `index.cjs`**.

- إذا شغّلت التطبيق من **node_app** فضع `.env` داخل:
  `/home/894422.cloudwaysapps.com/cmkdrtgqcv/node_app/.env`
- إذا شغّلته من **public_html** فضع `.env` داخل:
  `/home/894422.cloudwaysapps.com/cmkdrtgqcv/public_html/.env`

**المتغيرات المطلوبة لتسجيل الدخول:**

```env
NODE_ENV=production
SESSION_SECRET=سلسلة_عشوائية_طويلة_ومخفية
```

**لتسجيل الدخول بـ جوجل أضف (بنفس القيم التي في Google Cloud Console):**

```env
GOOGLE_CLIENT_ID=xxxxx.apps.googleusercontent.com
GOOGLE_CLIENT_SECRET=xxxxx
GOOGLE_CALLBACK_URL=https://sharfedu.com/api/auth/google/callback
```

- لا مسافات حول `=`.
- لا تضع القيم بين علامات اقتباس إلا إذا كانت تحتوي مسافات.

بعد تعديل `.env` أعد تشغيل التطبيق:

```bash
cd /home/894422.cloudwaysapps.com/cmkdrtgqcv/node_app
npx pm2 restart sharfedu
```

---

## 2) إضافة X-Forwarded-Proto في `.htaccess` (مهم للكوكي والتحويلات)

على السيرفر، في مجلد **public_html** (جذر الموقع)، تأكد أن ملف `.htaccess` يحتوي **قبل** قواعد RewriteRule على:

```apache
<IfModule mod_headers.c>
  RequestHeader set X-Forwarded-Proto "https"
</IfModule>
```

إذا لم تجدها، أضفها في بداية الملف ثم احفظ. هذا يخبر تطبيق Node أن الطلب الأصلي كان عبر HTTPS فيعمل الكوكي والتحويلات بشكل صحيح.

---

## 3) جوجل: تطابق رابط التحويل (Redirect URI)

في **Google Cloud Console** → تطبيقك → **APIs & Services** → **Credentials** → OAuth 2.0 Client:

- في **Authorized redirect URIs** يجب أن يكون بالضبط:
  `https://sharfedu.com/api/auth/google/callback`
- نفس القيمة في السيرفر في `.env`:
  `GOOGLE_CALLBACK_URL=https://sharfedu.com/api/auth/google/callback`

أي فرق (http بدل https، أو مسار مختلف) يسبب عدم عمل تسجيل الدخول بجوجل.

---

## 4) التحقق من عمل الـ API

من SSH:

```bash
curl -s https://sharfedu.com/api/auth/user
```

المتوقع قبل تسجيل الدخول: `null`.  
إذا ظهر خطأ أو اتصال مرفوض فالمشكلة في الوصول إلى Node (مثلاً الـ proxy أو المنفذ 5000).

---

## ملخص سريع

| الخطوة | أين |
|--------|-----|
| `.env` مع `NODE_ENV`, `SESSION_SECRET`, واختيارياً `GOOGLE_*` | مجلد تطبيق Node (node_app أو public_html) |
| `X-Forwarded-Proto` في `.htaccess` | public_html |
| Redirect URI مطابق في جوجل و `GOOGLE_CALLBACK_URL` | Google Console + .env |
| `npx pm2 restart sharfedu` بعد تغيير .env | SSH |
