# حل مشكلة تسجيل الدخول (التقليدي وجوجل) على السيرفر

إذا كان تسجيل الدخول لا يعمل (بريد/كلمة مرور أو جوجل)، تحقق من النقاط التالية على السيرفر.

---

## 1. ملف `.env` داخل مجلد تطبيق Node (مهم جداً)

على السيرفر، مجلد تطبيق Node (مثلاً **node_app**) يجب أن يحتوي على ملف **`.env`** بالقيم التالية. إن لم يكن موجوداً أنشئه يدوياً.

```env
NODE_ENV=production
PORT=5000
HOST=127.0.0.1

# ضروري لتسجيل الدخول — غيّره إلى سلسلة عشوائية طويلة
SESSION_SECRET=ضع-هنا-سلسلة-عشوائية-طويلة-وفريدة

# بريد الأدمن (للتسجيل الأول أو استعادة الأدمن)
ADMIN_EMAIL=بريدك@example.com

# قاعدة البيانات: استخدم مسار ملف sqlite.db داخل مجلد node_app
DATABASE_URL=file:sqlite.db
```

**لتسجيل الدخول عبر جوجل** أضف في نفس الملف:

```env
GOOGLE_CLIENT_ID=رقم-العميل-من-كونسول-جوجل
GOOGLE_CLIENT_SECRET=السر-من-كونسول-جوجل
GOOGLE_CALLBACK_URL=https://sharfedu.com/api/auth/google/callback
```

- في [Google Cloud Console](https://console.cloud.google.com/) → APIs & Services → Credentials: أنشئ OAuth 2.0 Client (نوع Web application).
- **Authorized redirect URIs** يجب أن يكون بالضبط: `https://sharfedu.com/api/auth/google/callback`.
- انسخ Client ID و Client Secret إلى `.env` كما أعلاه.

بعد تعديل `.env` أعد تشغيل التطبيق:

```bash
pm2 restart sharfedu
```

---

## 2. ملف `sqlite.db` وقاعدة البيانات

- يجب أن يكون ملف **sqlite.db** موجوداً **داخل مجلد node_app** على السيرفر (إما جاء مع الـ zip من البناء، أو ارفعه يدوياً).
- إذا كان الملف في مكان آخر، ضبط في `.env` مساره، مثلاً:
  ```env
  DATABASE_URL=file:/المسار/الكامل/إلى/sqlite.db
  ```
- إذا كان **sqlite.db** جديداً أو فارغاً، لن يكون هناك مستخدمين. أنشئ حساباً من صفحة التسجيل أو استخدم سكربتات المشروع لإنشاء أدمن (مثل `npm run migrate:admin` إن وُجدت).

---

## 3. ترويسة X-Forwarded-Proto (كوكي الجلسة)

عند تشغيل الموقع خلف Apache (مثل Cloudways)، يجب أن يعرف تطبيق Node أن الطلب جاء عبر **HTTPS** حتى يضبط كوكي الجلسة بشكل صحيح (Secure). لذلك تم إضافة السطر التالي في **.htaccess** في جذر الموقع (public_html):

```apache
RequestHeader set X-Forwarded-Proto "https"
```

- إذا قمت بنشر الموقع من جديد، تأكد أن ملف **.htaccess** المرفوع يحتوي على هذا السطر داخل بلوك توجيه `/api`.
- إذا لم يكن موجوداً، أضفه يدوياً في `.htaccess` على السيرفر داخل:
  `<IfModule mod_proxy.c>` قبل سطر `RewriteRule ^api...`

---

## 4. التحقق من عمل /api

- افتح في المتصفح: `https://sharfedu.com/api/auth/user`  
  - إن لم تكن مسجلاً: يتوقع أن ترى `null` أو رسالة من الـ API (وليس خطأ اتصال أو 502).
- إن ظهر خطأ اتصال أو 502: تأكد أن تطبيق Node يعمل (`pm2 list`) وأن **.htaccess** يوجّه `/api` إلى المنفذ 5000.

---

## 5. ملخص الخطوات على السيرفر

1. التأكد من وجود **.env** داخل **node_app** مع **SESSION_SECRET** و **DATABASE_URL=file:sqlite.db** و **GOOGLE_*** إن أردت الدخول بجوجل.
2. التأكد من وجود **sqlite.db** داخل **node_app** (أو مساره مضبوط في **DATABASE_URL**).
3. التأكد من أن **.htaccess** في **public_html** يحتوي على **RequestHeader set X-Forwarded-Proto "https"** داخل بلوك الـ proxy.
4. إعادة تشغيل التطبيق: **pm2 restart sharfedu**.
5. تجربة تسجيل الدخول مرة أخرى (بريد/كلمة مرور أو جوجل).

إذا استمرت المشكلة، راجع سجلات التطبيق:

```bash
pm2 logs sharfedu
```

وابحث عن رسائل مثل `[Auth]` أو أخطاء تتعلق بالجلسة أو قاعدة البيانات.
