# خطة عمل: تطابق المحلي مع السيرفر + نشر تلقائي

## الهدف النهائي

- **المحلي:** مصدر الكود (`src/`) → بناء (`npm run build`) → ناتج النشر (`deploy_public_html`). لا تغيير في هيكل المجلدات.
- **السيرفر:** نظيف ومعياري: `git_repo` = المستودع فقط، `public_html` = نسخة من محتوى `deploy_public_html` فقط (+ ملفات خاصة بالسيرفر).
- **الأتمتة:** عند قولك "اعكس التعديلات" → نرفع من المحلي → تنعكس على GitHub ثم على السيرفر ثم على الموقع تلقائياً.

---

## المرحلة 1 — تنظيف السيرفر وإعادة هيكلته (مرة واحدة عبر SSH)

**يمكن تنفيذ كل أوامر المرحلة 1 دفعة واحدة:** انسخ كتلة "أمر واحد للمرحلة 1" من نهاية هذا القسم والصقها في SSH.

---

### 1.1 نسخ احتياطي لما نحتاج الاحتفاظ به من public_html

على السيرفر (SSH):

```bash
APP_PATH="/home/894422.cloudwaysapps.com/cmkdrtgqcv"
cd "$APP_PATH/public_html"

# نسخ .env و .deploy-token خارج public_html (لنستعيدهما بعد التنظيف)
mkdir -p "$APP_PATH/backup_before_clean"
cp -p .env "$APP_PATH/backup_before_clean/.env" 2>/dev/null || true
cp -p .deploy-token "$APP_PATH/backup_before_clean/.deploy-token" 2>/dev/null || true
```

### 1.2 حذف كل محتويات public_html (ما عدا ما ننسخه من deploy_public_html لاحقاً)

نحذف من `public_html` كل شيء (مجلدات وملفات زائدة، مستودع، node_modules، إلخ):

```bash
APP_PATH="/home/894422.cloudwaysapps.com/cmkdrtgqcv"
cd "$APP_PATH/public_html"
# حذف كل المحتويات
find . -maxdepth 1 ! -name '.' ! -name '..' -exec rm -rf {} +
```

### 1.3 تحديث git_repo من GitHub ثم ملء public_html من deploy_public_html فقط

```bash
APP_PATH="/home/894422.cloudwaysapps.com/cmkdrtgqcv"
cd "$APP_PATH/git_repo"

git fetch origin
git reset --hard origin/main

# نسخ واجهة الموقع فقط من deploy_public_html إلى public_html
if [ -d deploy_public_html ] && [ -n "$(ls -A deploy_public_html 2>/dev/null)" ]; then
  cp -r deploy_public_html/* "$APP_PATH/public_html/"
  echo "Done: deploy_public_html copied to public_html"
else
  echo "ERROR: deploy_public_html missing or empty. Run npm run build locally and push."
fi
```

### 1.4 استعادة ملفات السيرفر الخاصة

```bash
APP_PATH="/home/894422.cloudwaysapps.com/cmkdrtgqcv"
cp -p "$APP_PATH/backup_before_clean/.env" "$APP_PATH/public_html/.env" 2>/dev/null || true
cp -p "$APP_PATH/backup_before_clean/.deploy-token" "$APP_PATH/public_html/.deploy-token" 2>/dev/null || true
chmod 600 "$APP_PATH/public_html/.deploy-token" 2>/dev/null || true
```

### 1.5 تشغيل تطبيق Node (API) من مكانه الصحيح

تطبيق الـ API (Node) يجب أن يعمل من **git_repo** وليس من public_html. مرة واحدة:

```bash
APP_PATH="/home/894422.cloudwaysapps.com/cmkdrtgqcv"
cd "$APP_PATH/git_repo"

# إيقاف أي عملية node قديمة
pkill -f "node.*index.cjs" 2>/dev/null || true
sleep 1

# تثبيت الحزم إن لم تكن موجودة
npm install

# تشغيل التطبيق (المنفذ 5000 للـ API)
mkdir -p logs
nohup env PORT=5000 node dist/index.cjs > logs/app.log 2>&1 &
echo "Node app started from git_repo. Check https://sharfedu.com"
```

**ملاحظة:** إذا كان `dist/index.cjs` غير موجود في `git_repo`، يلزم بناءه على السيرفر مرة (أو نضمن أن المستودع يرفع `dist` أو نضيف خطوة `npm run build` في السيرفر — انظر المرحلة 2).

## المرحلة 2 — التأكد أن git_repo فيه كل ما يلزم للنشر

- المحلي يرفع مع كل push: **`deploy_public_html`** (نتيجة `npm run build`) و **`server/public`** و **`src`** و **`scripts`**.
- على السيرفر لا ننفذ `npm run build`؛ نعتمد على **deploy_public_html** المرفوع في المستودع.
- تطبيق Node: إما نرفع **`dist/index.cjs`** في المستودع، أو نضيف في السكربت خطوة بناء على السيرفر (`npm run build` ثم تشغيل `node dist/index.cjs`). الخيار الأبسط: رفع **dist** مع الـ commit أو بناءه على السيرفر مرة بعد كل pull.

## المرحلة 3 — أتمتة "اعكس التعديلات"

### 3.1 عندك (المحلي)

1. تعديل في **`src/`** (أو أي ملف مصدر).
2. تشغيل **`npm run build`**.
3. إضافة ورفع:  
   `git add src deploy_public_html server/public scripts script` ثم  
   `git commit -m "وصف التعديل"` ثم  
   `git push origin main`.

### 3.2 على السيرفر (تلقائي)

- **ويب هوك:** عند كل push، GitHub Actions يستدعي عنوان الموقع (مثل `https://sharfedu.com/deploy-webhook.php?token=...`) فيضع ملف علم (مثلاً `deploy-requested.flag`) في **public_html**.
- **Cron (كل دقيقة):** يتحقق إن وُجد العلم، ثم:
  1. يدخل **git_repo**
  2. يسحب آخر التحديثات: `git fetch origin && git reset --hard origin/main`
  3. ينسخ **deploy_public_html** → **public_html** (مع الحفاظ على `.env` و `.deploy-token`)
  4. إن لزم: يعيد تشغيل تطبيق Node من **git_repo**
  5. يحذف ملف العلم

بهذا أي "اعكس التعديلات" (بعد push من المحلي) ينعكس على GitHub ثم على السيرفر ثم على الموقع خلال حوالي دقيقة.

### 3.3 سكربت النشر على السيرفر (محدّث)

سكربت واحد يشغّله الـ cron (أو تشغيله يدوياً عند الطلب):

- المسار: **git_repo/scripts/deploy-on-server.sh**
- يفعل: حفظ/استعادة `.env` → سحب من GitHub → نسخ **deploy_public_html** → **public_html** فقط → إعادة تشغيل Node من **git_repo**.

سنضمن أن هذا السكربت يستخدم المسار الثابت **cmkdrtgqcv** ولا يعتمد على وجود مستودع أو node داخل **public_html**.
