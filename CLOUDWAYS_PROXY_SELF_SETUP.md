# محاولة إعداد الـ Proxy بأنفسنا عبر SSH

اتبع الخطوات بالترتيب في الطرفية (SSH) على السيرفر.

---

## الخطوة 1: البحث عن ملف إعداد Nginx لتطبيقك

نفّذ هذه الأوامر واحداً تلو الآخر وأرسل **كل المخرجات** (أو صورة للطرفية).

```bash
ls -la /home/master/applications/cmkdrtgqcv/
```

```bash
find /home/master/applications/cmkdrtgqcv -name "*.conf" -o -name "*nginx*" 2>/dev/null
```

```bash
ls -la /etc/nginx/sites-enabled/ 2>/dev/null
```

```bash
ls -la /etc/nginx/conf.d/ 2>/dev/null
```

من النتيجة نعرف:
- إن وُجد ملف `.conf` أو شيء يشبه إعداد Nginx **داخل مجلد تطبيقك** (cmkdrtgqcv) وقد نستطيع تعديله.
- إن كان الإعداد فقط تحت `/etc/nginx/` ولا يسمح لك بالكتابة فيه، سنحتاج دعم Cloudways.

## الخطوة 2: (فقط إن وجدنا ملفاً قابلاً للتعديل)

سنعطيك نص الإعداد الذي نضيفه وأين نضعه، ثم أمر لإعادة تحميل Nginx — إن كان إعادة التحميل يتطلب sudo فستظهر رسالة صلاحيات وسنعتمد على الدعم.

نفّذ الخطوة 1 أولاً وأرسل المخرجات.
