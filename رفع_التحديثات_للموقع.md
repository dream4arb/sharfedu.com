# رفع التحديثات إلى الموقع (sharfedu.com)

تم البناء محلياً بنجاح. لِعكس التحديثات على الموقع اتبع التالي.

---

## 1) من جهازك (في مجلد المشروع)

### رفع التغييرات إلى GitHub

```bash
git add -A
git status
git commit -m "شارف التعليمية، فوتر، استعادة كلمة المرور، بناء الإنتاج"
git push origin main
```

(إن ظهرت ملفات لا تريد رفعها مثل `sqlite.db` أو ملفات شخصية، أزلهم من التتبع أو أضفهم لـ `.gitignore` ثم نفّذ `git add` و `commit` و `push` مرة أخرى.)

---

## 2) على السيرفر (Cloudways عبر SSH)

### سحب التحديثات وتشغيل التطبيق

```bash
cd "/home/master/applications/cmkdrtgqcv/git_repo"
# أو حسب مسار تطبيقك: cd "/home/894422.cloudwaysapps.com/cmkdrtgqcv/public_html/git_repo"

cp .env .env.backup 2>/dev/null || true
git fetch --all
git reset --hard origin/main
mv .env.backup .env 2>/dev/null || true

npm install
```

### نسخ واجهة الموقع إلى public_html (إن كان المشروع داخل public_html)

```bash
cp -r deploy_public_html/* ../public_html/ 2>/dev/null || cp -r server/public/* ../public_html/ 2>/dev/null || true
```

(اضبط المسار `../public_html` حسب مكان مجلد الدومين على سيرفرك.)

### إيقاف النسخة القديمة ثم تشغيل النسخة الجديدة

```bash
pkill -f "node dist/index.cjs" 2>/dev/null || true
nohup env PORT=5000 node dist/index.cjs > ../app.log 2>&1 &
```

أو إن كنت تستخدم PM2:

```bash
pm2 restart sharfedu
```
