# الحل النهائي للنشر التلقائي (بدون أوامر يدوية بعد كل تعديل)

بعد إعداد هذا **مرة واحدة**، كل **push** إلى `main` يحدّث الموقع تلقائياً خلال حوالي **دقيقة** دون تشغيل أي أوامر على السيرفر.

---

## كيف يعمل

1. ترفع التعديلات من جهازك: `git push origin main` (مع الملفات المبنية في الـ commit).
2. **GitHub Actions** يستدعي عنواناً على موقعك (ويب هوك) مع رمز سري.
3. السيرفر يضع ملف «علم» (flag).
4. **Cron** على السيرفر يشغّل كل دقيقة: إن وُجد الملف، يسحب من GitHub ويشغّل سكربت النشر ثم يحذف الملف.
5. الموقع يتحدّث خلال دقيقة من الـ push.

---

## الإعداد (مرة واحدة فقط)

### 1) إنشاء رمز سري للويب هوك

اختر سطراً عشوائياً طويلاً (مثلاً من https://randomkeygen.com أو نفّذ على جهازك):

```powershell
[Convert]::ToBase64String((1..32 | ForEach-Object { Get-Random -Maximum 256 }) -as [byte[]])
```

انسخ الناتج واحتفظ به.

### 2) على السيرفر عبر SSH

**أ) إنشاء ملف الرمز السري**

المسار: داخل **public_html** (جذر الموقع)، ملف اسمه **`.deploy-token`** ومحتواه سطر واحد = الرمز الذي أنشأته.

```bash
cd /home/894422.cloudwaysapps.com/cmkdrtgqcv/public_html
echo "الصق_هنا_الرمز_السري_بدون_اقتباس" > .deploy-token
chmod 600 .deploy-token
```

**ب) إضافة الـ Cron**

```bash
crontab -e
```

أضف السطر التالي (كل دقيقة) ثم احفظ واخرج:

```
* * * * * cd /home/894422.cloudwaysapps.com/cmkdrtgqcv/git_repo && [ -f ../public_html/deploy-requested.flag ] && git fetch origin && git reset --hard origin/main && bash scripts/deploy-on-server.sh && rm -f ../public_html/deploy-requested.flag
```

استبدل المسار `/home/894422.cloudwaysapps.com/cmkdrtgqcv` بمسار تطبيقك إن كان مختلفاً.

### 3) في GitHub — إضافة Secretين

- **Settings** → **Secrets and variables** → **Actions** → **New repository secret**

| الاسم | القيمة |
|--------|--------|
| `DEPLOY_WEBHOOK_URL` | `https://sharfedu.com/deploy-webhook.php` |
| `DEPLOY_WEBHOOK_TOKEN` | نفس الرمز الذي وضعته في `.deploy-token` على السيرفر |

## التأكد أن كل شيء يعمل

1. اعمل أي تعديل بسيط ثم ارفعه (مع البناء إن لزم): `git add -A` ثم `git commit -m "test"` ثم `git push origin main`.
2. افتح **GitHub** → **Actions** وتأكد أن الـ workflow ينجح (أخضر).
3. خلال حوالي **دقيقة** تحقق من الموقع — يفترض أن يظهر التعديل.

---

## ملخص

| ما تريده | ما تفعله |
|----------|----------|
| نشر تلقائي بعد كل push | أعد الإعداد أعلاه مرة واحدة |
| رفع تعديلات | من جهازك: `git push origin main` (مع الملفات المبنية) |
| لا أوامر على السيرفر | لا حاجة — الـ cron والويب هوك يتكفلان بالتحديث |

لا حاجة بعد اليوم لتشغيل `git pull` أو `deploy-on-server.sh` يدوياً.
