# إعداد لوحة تحكم الإدارة ومنظومة الهوية

## 1. التبعيات

```bash
npm install
```

تم إضافة `multer` و`passport-google-oauth20` إلى المشروع.

## 2. قاعدة البيانات (الترحيل)

قم بتشغيل أحد الأمرين:

```bash
npm run db:push
```

أو إن فشل (مثلاً على Windows):

```bash
npm run migrate:admin
```

ثم لتفعيل نظام إدارة المحتوى (CMS) والـ SEO:

```bash
npm run migrate:cms
```

هذا ينشئ جداول: `cms_stages`, `cms_subjects`, `cms_lessons`, `cms_content`, `seo_data`.

هذا يضيف حقول `role` و`google_id` لجدول المستخدمين، وينشئ جداول الإدارة: `admin_attachments`, `admin_lesson_html`, `admin_lesson_json`, `platform_stats`.

## 3. متغيرات البيئة (.env)

أضف أو حدّث في `.env`:

```env
# المدير الأول: البريد الذي يُمنح صلاحية admin عند التسجيل
ADMIN_EMAIL=admin@example.com

# Google OAuth (اختياري — لخيار «الدخول عبر جوجل»)
GOOGLE_CLIENT_ID=...
GOOGLE_CLIENT_SECRET=...
GOOGLE_CALLBACK_URL=http://localhost:5000/api/auth/google/callback
```

- **ADMIN_EMAIL**: أول حساب يُنشأ بهذا البريد (تسجيل جديد) يصبح مديراً تلقائياً.
- **Google OAuth**: إعداد مشروع في [Google Cloud Console](https://console.cloud.google.com/) وإضافة «معرف العميل» و«السر»، وتحديد عنوان إعادة التوجيه لـ `GOOGLE_CALLBACK_URL`.

## 4. إنشاء مدير

1. تأكد أن `ADMIN_EMAIL` مضبوط في `.env`.
2. سجّل حساباً جديداً من صفحة «إنشاء حساب» باستخدام نفس البريد.
3. سيُمنح هذا الحساب دور `admin` ويمكنه الدخول إلى `/admin`.

## 5. لوحة التحكم (/admin) - نظام إدارة محتوى (CMS)

- **الوصول**: فقط للمستخدمين المسجلين بدور `admin`.
- **الحماية**: Middleware يتحقق من تسجيل الدخول ثم من `role === 'admin'`.
- **المحتوى**:
  - **الرئيسية**: عدد الطلاب ونسبة الإنجاز.
  - **إضافة محتوى**: رفع ذكي مع قوائم مرتبطة (المرحلة → المادة → الفصل → الدرس) وتحديد التبويب (الدرس، الفيديو، الملخص، التعليم، الأسئلة). دعم PDF، روابط يوتيوب، كود HTML، وJSON. معاينة HTML.
  - **إدارة الأعضاء**: جدول مع بحث وتعديل وحذف. إحصائيات سريعة (الإجمالي، الجدد اليوم).
  - **إعدادات SEO**: اختيار أي صفحة وتعديل Meta Tags (Title, Description, Keywords) وحفظها في قاعدة البيانات لعرضها في Header.

## 6. الانعكاس الفوري في الدروس

- صفحة الدرس (مثل «زوايا المضلع») تطلب:
  - **محتوى التعليم**: `GET /api/content/lesson/:lessonId/education-html`
  - **الأسئلة**: `GET /api/content/lesson/:lessonId/json?key=questions`
- المصدر: قاعدة البيانات أولاً، ثم الملفات الثابتة تحت `attached_assets` كبديل.
- أي تحديث من لوحة التحكم (HTML أو JSON) يظهر في صفحات الدروس فوراً عند التحميل.

## 7. تسجيل الدخول وإنشاء الحساب

- **البريد + كلمة المرور**: مدعوم مع رسائل عربية.
- **الدخول عبر جوجل**: يعمل عند إعداد `GOOGLE_*` في `.env`؛ يظهر زر «الدخول عبر جوجل» في صفحة تسجيل الدخول.

## 8. روابط سريعة

| المسار        | الوصف                        |
|---------------|-----------------------------|
| `/login`      | تسجيل الدخول / إنشاء حساب  |
| `/dashboard`  | لوحة تحكم الطالب           |
| `/admin`      | لوحة تحكم الإدارة (مدير فقط) |
