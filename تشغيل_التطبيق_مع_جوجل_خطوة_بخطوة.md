# تشغيل التطبيق مع الدخول عبر جوجل — خطوة بخطوة

## ماذا نريد أن نفعل؟

نجعل السيرفر يشغّل تطبيق Node **مع** بيانات جوجل الجديدة (Client ID و Client Secret)، حتى يعمل زر «الدخول عبر جوجل» في الموقع.

---

## الخطوة 1 — فتح الاتصال بالسيرفر (SSH)

1. ادخل إلى السيرفر كما تعتاد (مثلاً من لوحة Cloudways أو من المتصفح أو برنامج SSH).
2. انتظر حتى يظهر سطر مثل: `[master_nyrmduupwf]:~$` أو `[master_nyrmduupwf]:public_html$`.
3. أنت الآن داخل الطرفية على السيرفر.

---

## الخطوة 2 — الدخول إلى مجلد التطبيق

اكتب الأمر التالي ثم اضغط **Enter**:

```bash
cd /home/894422.cloudwaysapps.com/cmkdrtgqcv/public_html
```

بعد التنفيذ يتغيّر السطر ليظهر فيه `public_html` (مثلاً: `[master_nyrmduupwf]:public_html$`). هذا يعني أنك داخل المجلد الصحيح.

---

## الخطوة 3 — تعيين المتغيرات ثم إعادة تشغيل التطبيق

انسخ **كل** الأسطر التالية من أول `export` إلى آخر `&` والصقها في الطرفية دفعة واحدة، ثم اضغط **Enter** مرة واحدة:

```bash
export NODE_ENV=production
export PORT=5000
export HOST=127.0.0.1
export SESSION_SECRET=sharfedu-secret-key-change-in-production-2026
export ADMIN_EMAIL=arb-33@hotmail.com
export DATABASE_URL=file:/home/894422.cloudwaysapps.com/cmkdrtgqcv/public_html/sqlite.db
export GOOGLE_CLIENT_ID=949610366276-emmp2cf50jp1gkgsgga8v38e0l12rk0j.apps.googleusercontent.com
export GOOGLE_CLIENT_SECRET=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
export GOOGLE_CALLBACK_URL=https://sharfedu.com/api/auth/google/callback
pkill -f "node index.cjs"
nohup node index.cjs > app.log 2>&1 &
```

**ماذا تفعل هذه الأوامر؟**

- **export ...** = تخبر السيرفر بقيم الإعدادات (منها بيانات جوجل الجديدة) لهذه الجلسة فقط.
- **pkill ...** = توقف أي تشغيل قديم للتطبيق.
- **nohup node index.cjs ...** = تشغيل التطبيق من جديد في الخلفية باستخدام الإعدادات التي عيّنتها (بما فيها Client ID و Client Secret الجديدان).

بعد التنفيذ قد يظهر رقم مثل `[1] 123456` — هذا طبيعي (معرّف العملية). لا تحتاج أن تكتب أي شيء آخر في الطرفية.

---

## الخطوة 4 — تجربة الدخول عبر جوجل من الموقع

1. افتح المتصفح وادخل إلى: **https://sharfedu.com/login**
2. اضغط على زر **«الدخول عبر جوجل»** (أو «Login via Google»).
3. اختر حساب جوجل وأكمل تسجيل الدخول.

إذا تمت العملية دون خطأ **redirect_uri_mismatch** أو «طلب غير صالح»، فمعناه أن التشغيل مع Client ID و Client Secret الجديدين يعمل.
