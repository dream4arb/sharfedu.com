# نشر الموقع من الصفر (بعد حذف كل شيء على السيرفر)

هذا الدليل لو ضبطت المشروع محلياً وقررت **حذف محتويات السيرفر ورفع كل شيء من جديد** بهيكل واضح.

---

## الهيكل بعد البناء

عند تشغيل **`npm run build`** يُنشأ مجلد **`for_server`** فيه:

| المجلد | المحتوى | أين يذهب على السيرفر |
|--------|---------|------------------------|
| **for_server/public_html/** | index.html، assets/، .htaccess، favicon، إلخ | **جذر الدومين** (المجلد الذي يخدم sharfedu.com، غالباً اسمه `public_html`) |
| **for_server/node_app/** | index.cjs، package.json، package-lock.json | مجلد منفصل لتشغيل تطبيق Node (مثلاً `node_app` بجانب أو داخل مسار التطبيق) |

---

## الخطوة 1: على جهازك

```bash
npm run build
```

بعدها ستجد:
- مجلد **for_server** وفيه **public_html** و **node_app**.
- في جذر المشروع أيضاً: **public_html_upload.zip** (محتوى الواجهة) و **node_app_upload.zip** (تطبيق Node) جاهزان للرفع. ارفعهما ثم فك الضغط في المكان المناسب على السيرفر.

---

## الخطوة 2: على السيرفر — حذف المحتوى القديم (إن رغبت)

- من **File Manager** أو **SFTP**: احذف محتويات مجلد جذر الدومين (مثلاً كل ما داخل `public_html`).
- إن كان عندك مجلد للتطبيق (مثل `git_repo` أو `node_app` قديم): احذف محتواه أو المجلد كاملاً حسب ما تريد البدء منه من الصفر.

---

## الخطوة 3: رفع المحتوى الجديد

### أ) جذر الموقع (الواجهة)

- ارفع **كل محتويات** `for_server/public_html/` إلى مجلد **جذر الدومين** على السيرفر (الذي يخدم sharfedu.com).
- النتيجة: في جذر الدومين يكون عندك مباشرة: `index.html`، مجلد `assets/`، `.htaccess`، `favicon.png`، إلخ (بدون مجلد وسيط اسمه public_html داخل الدومين).

### ب) تطبيق Node

- أنشئ مجلداً على السيرفر لتطبيق Node، مثلاً: **node_app** (يمكن أن يكون داخل مسار التطبيق، مثل: `public_html/../node_app` أو أي مسار تفضّله).
- ارفع **كل محتويات** `for_server/node_app/` إلى هذا المجلد.
- النتيجة: داخل **node_app** يكون عندك: `index.cjs`، `package.json`، `package-lock.json`.

---

## الخطوة 4: على السيرفر — إعداد وتشغيل Node

1. **الدخول لمجلد تطبيق Node:**
   ```bash
   cd /المسار/إلى/node_app
   ```
   (استبدل بالمسار الفعلي، مثلاً: `/home/894422.cloudwaysapps.com/cmkdrtgqcv/node_app`)

2. **إنشاء ملف .env** (إن لم يكن موجوداً):
   ```bash
   printf '%s\n' 'NODE_ENV=production' 'PORT=5000' 'HOST=127.0.0.1' 'SESSION_SECRET=سلسلة-عشوائية-طويلة' 'ADMIN_EMAIL=بريدك@example.com' > .env
   ```
   (عدّل SESSION_SECRET و ADMIN_EMAIL ثم احفظ.)

3. **تثبيت الحزم وتشغيل التطبيق:**
   ```bash
   npm install
   nohup env PORT=5000 node index.cjs > app.log 2>&1 &
   ```

4. تأكد أن **.htaccess** في جذر الدومين يوجّه **/api** إلى **127.0.0.1:5000** (المحتوى الموجود في `for_server/public_html/.htaccess` يفعل ذلك).

---

## الخطوة 5: التحقق

- افتح **https://sharfedu.com** (وليس عنوان IP أو منفذ 4200).
- يجب أن تظهر الواجهة وأن يعمل تسجيل الدخول وواجهة الـ API.

---

## ملخص المسارات على Cloudways (مثال)

- **جذر الدومين:**  
  `/home/894422.cloudwaysapps.com/cmkdrtgqcv/public_html/`  
  ← محتوى **for_server/public_html/** يرفع هنا (بحيث يكون index.html و assets داخل public_html مباشرة).

- **تطبيق Node:**  
  `/home/894422.cloudwaysapps.com/cmkdrtgqcv/node_app/`  
  ← محتوى **for_server/node_app/** يرفع هنا، ومنه تُنفَّذ أوامر `npm install` و `node index.cjs`.

بهذا يكون كل مجلد في مكانه الصحيح ويمكنك إعادة النشر من الصفر في أي وقت بنفس الخطوات.
