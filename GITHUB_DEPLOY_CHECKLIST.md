# خطوات ما بعد ربط المشروع بـ GitHub

بما أنك رفعت الملفات على GitHub وعملت الربط، هذه الخطوات التي تحتاجها حتى يعمل الموقع فعلياً.

**لأتمتة النشر (Auto-Deploy) من GitHub إلى Cloudways:** راجع **[CLOUDWAYS_AUTO_DEPLOY.md](CLOUDWAYS_AUTO_DEPLOY.md)** — يشرح Webhooks من لوحة Cloudways أو استخدام GitHub Actions.

---

## 1. إعدادات يجب ضبطها على المنصة (Cloudways أو غيرها)

عند النشر من Git، المنصة تحتاج أن تعرف **كيف تبني** و **كيف تشغّل** المشروع.

| الإعداد | القيمة |
|--------|--------|
| **أمر البناء (Build Command)** | `npm ci && npm run build` |
| **أمر التشغيل (Start Command)** | `npm start` |
| **مجلد الجذر** | جذر المشروع (لا تضع `client` أو غيره) |

---

## 2. متغيرات البيئة (إن وُجدت)

إذا كانت المنصة تسمح بإضافة Environment Variables أضف:

| المتغير | الوصف | مثال |
|---------|--------|------|
| `NODE_ENV` | للإنتاج | `production` |
| `PORT` | المنفذ (غالباً تضعه المنصة تلقائياً) | `5000` أو القيمة التي تعطيك إياها المنصة |
| `HOST` | على Railway / Render أضف إن لم يعمل الموقع | `0.0.0.0` |

إذا كنت تستخدم قاعدة بيانات أو API keys (مثل Gemini)، أضفها أيضاً في متغيرات البيئة.

---

## 3. على Cloudways تحديداً (بعد الربط مع GitHub)

1. **التأكد من أن التطبيق من نوع Node.js** وليس PHP (أو إنشاء تطبيق Node جديد مربوط بنفس الريبو).
2. **تعيين أمر البناء:**  
   `npm ci && npm run build`
3. **تعيين أمر التشغيل:**  
   `npm start`  
   أو إن طلبت المنصة مسار الملف:  
   `node dist/index.cjs`
4. **تشغيل النشر (Deploy)** من لوحة Cloudways بعد الربط مع GitHub.
5. إذا استمرت المشكلة، راجع [دليل Node.js على Cloudways](https://www.cloudways.com/blog/how-to-host-a-node-js-application/).

---

## 4. التحقق بعد النشر

- افتح رابط التطبيق الذي تعطيك إياه المنصة (مثل `https://....cloudwaysapps.com` أو الدومين الذي ربطته).
- يفترض أن تظهر **منصة شارف** وليس صفحة PHP أو صفحة فارغة.
- إذا ظهر خطأ، راجع **سجلات التشغيل (Logs)** في لوحة الاستضافة لمعرفة سبب الفشل.

---

## ملخص الأوامر (للمرجع)

```bash
# البناء
npm ci
npm run build

# التشغيل
npm start
# أو
PORT=5000 node dist/index.cjs
```

بعد تنفيذ هذه الخطوات على المنصة التي ربطتها بـ GitHub، يفترض أن يعمل الموقع بشكل صحيح.
